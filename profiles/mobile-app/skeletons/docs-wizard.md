# Wizard: Docs

> **Profile:** Mobile App

## When to Use
Documentation maintenance.

## Screen Documentation Template

```markdown
# Screen: {Name}

## Purpose
What this screen does.

## Route
`/{route-name}`

## Widgets/Components
| Widget | Purpose |
|--------|---------|
| {name} | {description} |

## State
| Variable | Type | Description |
|----------|------|-------------|
| {name} | {type} | {desc} |

## Events
| Event | Description |
|-------|-------------|
| {name} | {description} |

## Platform-Specific
- iOS: {specifics}
- Android: {specifics}

## Accessibility
- Screen reader labels
- Touch targets (min 48x48)
- Color contrast
```

## Customization
Add user flow diagrams, onboarding guides.
